# 新スキル追加チェックリスト（テンプレ駆動）

このチェックリストの目的は、
**無駄なスキル増殖を防ぎつつ、必要な時に確実に使えるスキルだけを増やす** ことです。

> **重要**: スキルは `.claude/skills/` に配置し、Proxy の skill registry 経由で運用します。
> Claude 側の MCP 露出は増やさず、`skill_search` / `skill_run` 経由でアクセスします。
> 詳細: [26_PROXY_SINGLE_ENTRYPOINT.md](./26_PROXY_SINGLE_ENTRYPOINT.md)

## A. 追加前（重複チェック）
- [ ] 既存スキルで代替できないか？
- [ ] "名前が違うだけの同じスキル" を作ろうとしていないか？
- [ ] そのスキルは「いつ使うか」が明確か？

## B. テンプレ選択（精度を上げる）
- [ ] Docsサイト用 / EC用 / 社内ツール用 など、該当テンプレを選ぶ
- [ ] テンプレがない場合は、まずテンプレを作る（次回のため）

## C. スキル仕様（最低限これを書く）
- [ ] 目的（何をできるようにするか）
- [ ] 入力（何を渡すか：URL、検索語、IDなど）
- [ ] 出力（何が返るか：要約、JSON、ファイル、リンクなど）
- [ ] 失敗時（どうなるか、どうリカバーするか）
- [ ] 危ない操作の有無（あるなら承認ルール）

## D. コンテキスト節約（絶対）
- [ ] 長文は会話に貼らず、メモリへ逃がす
- [ ] 会話には要点（結論/次アクション/参照ID）だけ残す
- [ ] MCPツールはスキルの中で呼ぶ（Claude側露出を増やさない）

## E. 動作確認
- [ ] 最小入力で成功することを確認
- [ ] 例外ケース（URLが404、ログイン必要等）でも壊れない
- [ ] `npm test / lint / typecheck` が通る（必要なら）

## F. ログ・運用
- [ ] Issueを作って開始している（テンプレ使用）
- [ ] PRにIssue番号をリンク
- [ ] マージ後、Issueに結果（使い方/注意点）を追記
